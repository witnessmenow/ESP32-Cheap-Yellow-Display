Happy Pi Day!

A Monte Carlo "dart throwing" approximation of pi. https://en.wikipedia.org/wiki/Monte_Carlo_integration

Stores progress in NVRAM so it can resume on reset.

Still converges on a slightly inaccurate value for pi. After ~8G "throws" I get a value of 3.141534. For a kilometer-wide circle, the calculated circumference using this wrong value would be off by 59mm.

There are much better ways to find pi but hey, who doesn't love throwing darts?
